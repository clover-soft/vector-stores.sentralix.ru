### Шаг 1: Каркас FastAPI + Docker + логирование
- Цель: 
  - Поднять сервис FastAPI в Docker, убедиться что сервис доступен и пишет логи согласно конфигурации.
- Действия: 
  - Реализовать точку входа `main.py` и создать FastAPI-приложение.
  - Подключить единый модуль/класс логирования (формат/уровень/файл/вывод в консоль — из env).
  - Добавить `config.py` для чтения переменных окружения (никакого хардкода).
  - Подключить и описать используемые переменные окружения:
    - `ALLOW_HOSTS` — список разрешённых хостов/IP (через запятую) для ограничения входящих запросов (при наличии middleware).
    - `LOG_LEVEL` — уровень логирования (например `DEBUG`, `INFO`, `WARNING`, `ERROR`).
    - `LOG_FORMAT` — формат строки логов (например с `request_id`).
    - `LOG_FILE` — путь к файлу логов внутри контейнера (например `/var/log/vector-store-log.log`).
    - `DATABASE_URI` — строка подключения к MariaDB (SQLAlchemy), нужна для дальнейших шагов и может использоваться уже на этапе health-проверок БД.
    - `YC_FOLDER_ID` — идентификатор каталога Yandex Cloud (понадобится на шаге интеграции провайдера `yandex`).
    - `YC_SA_KEY_JSON_B64` — base64-строка с JSON ключом сервисного аккаунта Yandex Cloud.
  - Добавить базовую ручку `GET /health`.
  - Убедиться, что сервис корректно стартует в Docker через текущие `Dockerfile` и `docker-compose.yaml`.
- Ожидаемый результат: 
  - Контейнер стартует без ошибок, `GET /health` возвращает успешный ответ, Swagger доступен, логи пишутся.
- Критерии готовности (Definition of Done):  
  - `docker-compose up` поднимает сервис.
  - `GET /health` возвращает `200`.
  - Логи создаются/обновляются в пути, заданном `LOG_FILE` (либо выводятся в stdout при `LOG_TO_CONSOLE=true`).
  - Конфигурация читается из env через `config.py`.
- Выполнено: в процессе
