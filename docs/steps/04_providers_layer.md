### Шаг 4: Слой провайдеров (yandex/openai/sentralix) + проверка индексов
- Цель: 
  - Реализовать слой интеграций с провайдерами и единый контракт фасада, добавить проверку текущих индексов у провайдера.
- Действия: 
  - Используемые переменные окружения:
    - `YC_FOLDER_ID` — идентификатор каталога Yandex Cloud. Используется провайдером `yandex`.
    - `YC_SA_KEY_JSON_B64` — base64-строка с JSON ключом сервисного аккаунта Yandex Cloud. Используется провайдером `yandex`.
  - Добавить общий контракт провайдера (интерфейс/базовый класс), который реализуют все провайдеры.
  - Создать интеграции в отдельных папках:
    - `providers/yandex/`
    - `providers/openai/`
    - `providers/sentralix/`
  - Реализовать сервис-роутер/фабрику выбора провайдера по `provider_type`.
  - Реализовать ручку:
    - `GET /api/v1/providers/{provider_type}/indexes`
  - Настроить конфигурацию провайдеров через env (`YC_*`, `OPENAI_API_KEY`, и т.д.).
  - Обработку ошибок провайдера сделать единообразной (логирование + корректные HTTP ответы).
- Ожидаемый результат: 
  - Сервис умеет обращаться к провайдеру и получать список текущих индексов в провайдере по единому контракту.
- Критерии готовности (Definition of Done):  
  - Все провайдеры реализуют единый контракт.
  - Ручка `GET /providers/{provider_type}/indexes` возвращает данные и корректно обрабатывает ошибки.
  - Конфигурация интеграций полностью вынесена в env.
  - Swagger документирован (описания, примеры, коды ответов).
- Выполнено: нет
