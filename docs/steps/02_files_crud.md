### Шаг 2: CRUD файлов (доменные файлы)
- Цель: 
  - Реализовать управление файлами доменных баз знаний: загрузка, хранение, получение метаданных, изменение метаданных, удаление.
- Действия: 
  - Используемые переменные окружения:
    - `DATABASE_URI` — строка подключения к MariaDB (SQLAlchemy). Используется сервисным слоем для CRUD операций с `rag_files`.
  - Реализовать SQLAlchemy модель для таблицы `rag_files` согласно DDL.
  - Добавить сервисный слой для CRUD операций с файлами.
  - Реализовать API ручки:
    - `POST /api/v1/files` (multipart upload)
    - `GET /api/v1/files`
    - `GET /api/v1/files/{file_id}`
    - `GET /api/v1/files/{file_id}/download`
    - `PATCH /api/v1/files/{file_id}`
    - `DELETE /api/v1/files/{file_id}`
  - Организовать хранение файлов в контейнере в `/files/<domain_id>/...`.
  - Обеспечить доменную изоляцию:
    - `domain_id` обязателен для всех операций;
    - все выборки/изменения строго фильтруются по `domain_id`.
  - Заполнять `local_path`, `size_bytes`, `file_type` (и при необходимости `tags`, `notes`).
  - Поддержать поле `chunking_strategy` для OpenAI (JSON, опционально).
- Ожидаемый результат: 
  - Внешний бэкенд может загружать и управлять файлами домена; файлы физически лежат в доменной папке.
- Критерии готовности (Definition of Done):  
  - Загрузка файла создаёт запись в `rag_files` и сохраняет файл на диск.
  - Список/получение метаданных возвращает только данные текущего `domain_id`.
  - Удаление удаляет запись и физический файл.
  - Swagger документирован (описания, схемы, коды ответов).
- Выполнено: в процессе
