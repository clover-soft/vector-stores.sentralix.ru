### Шаги разработки: оглавление
- Цель: 
  - Зафиксировать план реализации проекта по шагам и обеспечить удобную навигацию.
- Действия: 
  - Использовать данный файл как индекс для документов в `docs/steps/`.
  - По мере выполнения шагов обновлять поле «Выполнено» в соответствующем файле шага.
- Ожидаемый результат: 
  - Быстрая навигация по плану реализации и единый источник правды по статусам выполнения.
- Критерии готовности (Definition of Done):  
  - Все шаги перечислены и содержат ссылки на соответствующие файлы.
- Выполнено: да

## Список шагов

1. [Шаг 1: Каркас FastAPI + Docker + логирование](./01_fastapi_skeleton.md)
   - Поднять сервис в Docker, реализовать `/health`, настроить логирование и конфиг через env.

2. [Шаг 2: CRUD файлов (доменные файлы)](./02_files_crud.md)
   - Загрузка/скачивание/метаданные/удаление, хранение в `/files/<domain_id>/...`, доменная изоляция.

3. [Шаг 3: CRUD индексов + связь индекс↔файл](./03_indexes_crud_and_binding.md)
   - CRUD индексов в локальной БД, привязка/отвязка файлов к индексу.

4. [Шаг 4: Слой провайдеров (yandex/openai/sentralix) + проверка индексов](./04_providers_layer.md)
   - Единый контракт провайдера, фасады интеграций, ручка проверки индексов у провайдера.

5. [Шаг 5: Провайдер Yandex (OpenAI-compat) + полный контракт Vector Stores API через `openai`](./05_yandex_provider_vector_stores.md)
   - Реализация провайдера `yandex` через `openai` SDK и сервисный фасад для всех операций Vector Stores.

10. [Шаг 10: Отложенная индексация (полный цикл)](./10_deferred_indexing_pipeline.md)
    - Создание индекса у провайдера, получение `external_id`, прикрепление файлов, запуск/опрос статуса, синхронизация статусов в БД.
